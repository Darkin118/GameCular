# 📦 개발 과정 아카이브

이 폴더는 GameCular 프로젝트의 개발 과정을 보여주는 초기 버전 파일들입니다.

---

## 📅 개발 타임라인

### v1.0 - 초기 구조 (파일 분리)

**파일 구성**:
- `v1-index.html`: 기본 HTML 구조
- `styles.css`: 스타일시트 분리
- `script.js`: JavaScript 로직 분리
- `games.json`: 정적 더미 데이터

**특징**:
- 전통적인 파일 분리 구조
- 정적 JSON 데이터 사용
- 기본적인 UI/UX
- 보라색 테마

---

### v2.0 - 최종 버전 (통합 + 우주 테마)

**위치**: 루트의 `index.html`

**주요 변경사항**:
- 모든 코드를 단일 HTML 파일로 통합
- RAWG API 실시간 연동
- 우주 망원경 테마 적용
- 애니메이션 별 효과
- 글래스모피즘 UI
- 향상된 에러 처리
- 중복 게임 추천 방지

---

## 🔄 주요 변경사항 비교

| 항목 | v1.0 | v2.0 |
|------|------|------|
| **파일 구조** | 분리 (4개) | 통합 (1개) |
| **데이터 소스** | 정적 JSON | RAWG API |
| **테마** | 보라색 단색 | 우주 망원경 |
| **배경** | 그라디언트 | 애니메이션 별 |
| **스타일링** | 기본 CSS | 글래스모피즘 |
| **에러 처리** | 최소한 | 폴백 시스템 |
| **로딩 상태** | 없음 | 있음 |
| **게임 정보** | 제한적 | 상세 (평점, 플랫폼) |
| **중복 방지** | 없음 | 있음 |
| **반응형** | 부분적 | 완전 |

## 📖 학습 포인트

### 1. 파일 분리 vs 통합
- **v1.0**: 전통적인 분리 구조로 시작
- **v2.0**: 소규모 프로젝트의 장점을 살려 단일 파일로 통합
- **배운 점**: 배포 및 유지보수 관점에서의 트레이드오프 이해

### 2. 정적 데이터 vs API
- **v1.0**: JSON 파일로 프로토타입 빠르게 구현
- **v2.0**: 실제 API 연동으로 실전 경험
- **배운 점**: 비동기 처리 및 에러 핸들링의 중요성

### 3. 디자인 진화
- **v1.0**: 기본적인 보라색 테마
- **v2.0**: 우주를 컨셉으로 한 몰입형 경험
- **배운 점**: 테마가 사용자 경험에 미치는 영향

### 4. 점진적 개선
- 기본 기능 → 완성도 높은 제품
- 사용자 피드백 반영
- 지속적인 리팩토링

## 💡 v1.0 파일 사용법

### 실행 방법
```bash
# 1. v1-index.html 파일명 변경
mv v1-index.html index.html

# 2. 같은 폴더에 styles.css, script.js, games.json 위치

# 3. 브라우저로 열기
open index.html
```

### 주의사항
- 파일들이 같은 폴더에 있어야 함
- API 연동 없이 더미 데이터만 사용
- 최신 기능 없음 (참고용)

## 🎯 이 아카이브의 가치

이 파일들은 단순한 백업이 아닙니다:

1. **학습 여정의 증거**: 어떻게 발전했는지 보여줌

3. **교육 자료**: 다른 학습자들을 위한 참고 자료
4. **포트폴리오**: 문제 해결 능력과 성장 과정 입증
