# 📦 개발 과정 아카이브

이 폴더는 GameCular 프로젝트의 개발 과정을 보여주는 초기 버전 파일들입니다.

## 📅 개발 타임라인

### v1.0 - 초기 구조 (파일 분리)
- `styles.css`: 스타일시트 분리
- `script.js`: JavaScript 로직 분리

**특징:**
- 전통적인 파일 분리 구조
- 정적 JSON 데이터 사용
- 기본적인 UI/UX

### v2.0 - 최종 버전 (통합 + API 연동)
- 위치: 루트의 `index.html`
- 모든 코드를 단일 HTML 파일로 통합
- RAWG API 실시간 연동
- 향상된 UI/UX 및 에러 처리

## 🔄 주요 변경사항

| 항목 | v1.0 | v2.0 |
|------|------|------|
| 파일 구조 | 분리 (3개) | 통합 (1개) |
| 데이터 소스 | 정적 JSON | RAWG API |
| 스타일링 | 기본 CSS | 그라디언트 + 애니메이션 |
| 에러 처리 | 최소한 | 완전한 처리 |
| 로딩 상태 | 없음 | 있음 |
| 게임 정보 | 제한적 | 상세 (평점, 플랫폼 등) |

## 📖 학습 포인트

1. **파일 분리 vs 통합**
   - 소규모 프로젝트에서 단일 파일의 장점 이해
   - 배포 및 유지보수 관점에서의 트레이드오프

2. **정적 데이터 vs API**
   - API 연동의 실전 경험
   - 비동기 처리 및 에러 핸들링

3. **점진적 개선**
   - 기본 기능 → 완성도 높은 제품
   - UX 개선 과정

---

이 파일들은 **교육 및 참고 목적**으로 보관되며, 실제 사용은 루트의 `index.html`을 권장합니다.
